# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

PSEye — a free, community-first tracker for the Philippine Stock Exchange (PSE). pnpm + Turborepo monorepo.

## Commands

Run from the repo root unless noted.

- `pnpm install` — install all workspace deps
- `pnpm dev` — `turbo run dev` (starts `apps/web` on localhost:3000)
- `pnpm build` / `pnpm lint` / `pnpm typecheck` — run across all packages via Turbo (each depends on `^build` first)
- Scope any script to one package: `pnpm --filter @pseye/web <script>` (package names: `@pseye/web`, `@pseye/db`, `@pseye/etl`, `@pseye/source-quotes`, `@pseye/source-news`, `@pseye/source-corporate-actions`, `@pseye/source-block-sales`, `@pseye/source-disclosures`, `@pseye/source-offerings`, `@pseye/source-foreign-flow`, `@pseye/treemap-layout`, `@pseye/config`)
- `pnpm db:generate` / `pnpm db:push` — Drizzle Kit generate/push against `packages/db/src/schema.ts`. `generate` doesn't need `DATABASE_URL` (diffs against `packages/db/drizzle/`'s migration history); `push` does. Migrations exist through `0004_*.sql` but have never been pushed to a real database — run `pnpm db:push` once a real `DATABASE_URL` is available, before the ETL jobs or the DB-backed read path below will do anything.
- `pnpm --filter @pseye/etl fetch-quotes` / `fetch-news` / `fetch-corporate-actions` / `fetch-block-sales` / `fetch-disclosures` / `fetch-offerings` / `fetch-foreign-flow` / `fetch-historical-quotes` / `fetch-market-snapshot` / `backfill-company-profiles` / `migrate-db` — run an ETL job locally; all require `DATABASE_URL`. In production these run on a schedule via GitHub Actions (`.github/workflows/*.yml` — hourly for quotes/news/disclosures/market-snapshot, daily for corporate-actions/offerings/historical-quotes, weekly for foreign-flow, monthly for block-sales), not on app request. `backfill-company-profiles` and `migrate-db` are one-off/manual, no workflow schedule (see their own doc comments).
- `pnpm test` — `turbo run test`. Has real coverage now: `@pseye/source-quotes` (PSE Edge stock-data/chart-data HTML+JSON parsers, `PseEdgeQuoteSource` error handling), `@pseye/source-disclosures` and `@pseye/source-corporate-actions` (their PSE Edge HTML parsers), and `@pseye/etl` (the foreign-flow PDF parser — see below). Scope with `pnpm --filter <pkg> test`.

## Architecture

Turborepo/pnpm workspaces: `apps/*`, `packages/*`, `packages/sources/*`, `etl`.

- **`apps/web`** — Next.js 16 (App Router) + React 19 frontend. Routes: `/` (`MarketMapPage`, the treemap "market map"), `/charts` (NASDAQ-only TradingView embed — see below), `/news`, `/calendar` (dividend & corporate actions), `/block-sales`, `/disclosures`, `/offerings` (IPO/follow-on tracker), `/foreign-flow`, and `/dca` (cost-averaging calculator). All are async Server Components using ISR (`export const revalidate`) rather than client fetching, except the DCA calculator's interactive core (`DcaCalculator`/`DcaChart`, client components) and the Charts page's TradingView widget (also client-side, third-party script). The DCA calculator now has a real API route — see `app/api/history/route.ts` below — it isn't purely in-browser anymore. `apps/web/AGENTS.md` / `CLAUDE.md` flags that this Next.js version has breaking changes vs. training data — check `node_modules/next/dist/docs/` before writing Next-specific code.
- **`/charts`** (`app/charts/page.tsx`, `components/TradingViewChart.tsx`) — embeds TradingView's free "Advanced Chart" widget. **TradingView's free embeddable widgets refuse to serve PSE-listed symbol data** — confirmed live across multiple widget types (Advanced Chart, Symbol Overview, Single Quote) and multiple PSE tickers plus the PSEi index; all return "This symbol is only available on TradingView" while the identical widget renders real data fine for e.g. `NASDAQ:AAPL`. Don't re-attempt PSE symbols in a TradingView widget expecting a config fix — it won't work. The ticker picker is scoped to a curated NASDAQ list instead; `allow_symbol_change` on the widget still lets a visitor search any other TradingView-supported symbol themselves. Also: TradingView's script overwrites `style.height="100%"` (inline, beats any Tailwind class) on the element carrying the `tradingview-widget-container` class — give a wrapper *outside* that class the fixed height, or the widget collapses to a tiny default size.
- **`apps/web/lib/*.ts`** — the "DB-backed when `DATABASE_URL` is set and populated, otherwise `Mock*Source`, falling back on any DB error too" pattern is now used by several files, not just quotes — keep this one-fallback-path-per-feature shape rather than instantiating a `Mock*Source`/real source directly in a page: `quotes.ts` (`getDailyQuotes`), `disclosures.ts` (`getDisclosures`), `corporateActions.ts` (`getCorporateActions`), `foreignFlow.ts` (`getForeignFlowPageData` — index-level and per-stock fall back *independently*, since only index-level foreign flow is real; mixing "real index chart + fake per-stock table" is intentional, not a bug), and `historicalQuotes.ts` (`getHistoricalQuotes` — falls back for *all* requested tickers together, not per-ticker, since blending a real close series with a synthetic one in the same composite average would be misleading). `getDailyQuotes()`'s `Quote.price`/`pctChange` are `number | null` — null means PSE Edge had no trade to report (suspended ticker, no fill yet today); render that as "N/A", never coerce to 0.
- **`apps/web/app/api/history/route.ts`** — the app's first (and so far only) Route Handler. Backs the DCA calculator: `DcaCalculator.tsx` can't call a real `HistoricalQuoteSource` directly the way it called `MockHistoricalQuoteSource` before (no CORS to PSE Edge, and it would mean live-scraping per keystroke instead of the batch-job-writes-to-DB pattern every other feature uses) — so it `fetch()`es this route instead, which reads `historicalQuotes.ts` (our own DB, or mock fallback) server-side. If a future interactive client feature needs real per-request data, follow this same shape rather than reaching for a `*Source` from client code.
- **`packages/treemap-layout`** (`@pseye/treemap-layout`) — pure layout + color math, no React/DOM dependency (kept that way so it can be reused by a planned static Satori-rendered share image, per its own doc comments). `computeLayout.ts` groups stocks by sector and sizes boxes by market cap with `d3-hierarchy`'s squarified treemap. `color.ts` builds the diverging (red↓ / green↑) `%`-change color scale with `d3-scale`/`d3-interpolate`, separately for light/dark mode, plus a WCAG-contrast helper (`getContrastText`) to pick label ink color. `ForeignFlowChart` reuses the same green/red finance convention for its own polarity metric (net buying/selling).
- **`packages/db`** (`@pseye/db`) — Drizzle ORM schema (`daily_quotes`, `news_items`, `corporate_actions`, `block_sales`, `disclosures`, `offerings`, `index_foreign_flow`, `stock_foreign_flow`, `historical_quotes`, `market_snapshot`, `company_profiles`) + Neon serverless Postgres client (`createDb`) + hand-written query helpers (`queries.ts`, e.g. `getLatestDailyQuotes`, `getHistoricalQuotes`, `getIndexForeignFlowHistory`). `drizzle.config.ts` reads `DATABASE_URL` — but `drizzle-kit generate` itself doesn't need a live DB (it diffs `schema.ts` against `packages/db/drizzle/`'s existing migration snapshots), so schema changes can be made and migrations generated even without `DATABASE_URL` set. Migrations exist through `0004_*.sql` but have never been pushed to a real database — run `pnpm db:push` once a real `DATABASE_URL` is available, before the ETL jobs or any DB-backed read path will do anything. `daily_quotes.price`/`pct_change` are nullable — see `apps/web/lib/quotes.ts` above.
- **`packages/sources/*`** — one pluggable source interface per feature: `quotes` (`QuoteSource`, `HistoricalQuoteSource`), `news` (`NewsSource`), `corporate-actions` (`CorporateActionSource`), `block-sales` (`BlockSaleSource`), `disclosures` (`DisclosureSource`), `offerings` (`OfferingSource`), `foreign-flow` (`ForeignFlowSource`). Swap implementations behind the interface; don't change callers.
  - **Real, PSE Edge–backed** (same knowing legal tradeoff as each other — see docs/PLANNING.md's Open Question #1 — public pages/endpoints, not a licensed feed; each runs from a scheduled ETL job only, never called per-request from the web app): RSS news (`createRssSource`); **`quotes`'s `PseEdgeQuoteSource`** (`packages/sources/quotes/src/pseEdge/`) — per-company "Stock Data" pages (`stockData.do?cmpy_id=...`) for delayed/EOD price, % change, market cap; **`quotes`'s `PseEdgeHistoricalQuoteSource`** (same dir) — real daily OHLC via `/common/DisclosureCht.ax` (the JSON endpoint that page's own price chart calls; needs a `security_id` read off that same page's dropdown first, not just `cmpy_id`); **`disclosures`'s `PseEdgeDisclosureSource`** (`packages/sources/disclosures/src/pseEdge/`) — POSTs `/announcements/search.ax`, matches rows to our roster by `cmpy_id`; **`corporate-actions`'s `PseEdgeCorporateActionSource`** (`packages/sources/corporate-actions/src/pseEdge/`) — POSTs `dividends_and_rights_info_list.ax`, Dividends tab only (Rights tab is mostly "TBA" placeholder dates, not worth mapping yet). All of these key off `@pseye/source-quotes`'s static `PSE_EDGE_COMPANIES` ticker→`cmpy_id` roster (`pseEdgeCompanyDirectory.ts`), matching by `cmpy_id`/`security_id` rather than by company-name text (PSE Edge's own name strings occasionally differ in punctuation/casing).
  - **Real, but not PSE Edge — a PDF, not HTML**: index-level foreign flow. See the `etl` bullet below — this one deliberately does *not* live in `packages/sources/foreign-flow`.
  - **Still `Mock*Source` placeholders**, each investigated this project and found to have no free public source (don't re-investigate without new information): `block-sales`'s `MockBlockSaleSource` and `foreign-flow`'s per-stock top-buying/top-selling half of `MockForeignFlowSource` — both live in PSE's full Monthly Report, which is *not* freely published; `pse.com.ph/market-report/` only links a "Preview" (cover page + written review, no data tables) of it. `offerings`'s `MockOfferingSource` — the one free real feed found (`disclosureData/listing_applicants_list.do`, "Listing Applicants") is currently empty ("no data.") and has no subscription-window dates at all, which the `offerings` DB table requires (`subscription_start`/`end` are `NOT NULL` and part of its unique key) — wiring it in for real would need a schema change to serve a source that's empty in production anyway.
  - `MockHistoricalQuoteSource` takes the *current* quotes as a constructor param (not a fixed internal list) so its anchor price matches whatever `QuoteSource` the caller is using (DB or mock) — see `apps/web/lib/historicalQuotes.ts`. Its synthetic close is floored at ₱0.01 — a sub-centavo mock price (there's a ₱0.012 sample ticker) could otherwise round to an exact 0.00 and divide-by-zero in `simulateDca`/`buildCompositeHistory`.
- **`etl`** — standalone `tsx` scripts (no build step) invoked by the GitHub Actions workflows in `.github/workflows/` (one per job; cadence matches each real-world source's publish schedule — hourly during PSE trading hours for quotes, hourly for news/disclosures/market-snapshot, daily for corporate-actions/offerings/historical-quotes, weekly for foreign-flow, monthly for block-sales); each upserts source data into Postgres via `@pseye/db`, deduping on a natural key per table (ticker+date, URL, reference number, etc. — see each job for its `onConflictDoNothing`/`onConflictDoUpdate` target).
  - **`etl/lib/pseMarketWatch/`** — `PseMarketWatchForeignFlowSource` (used only by `fetch-foreign-flow.ts`) extracts real index-level Foreign Buying/Selling from PSE's free weekly "Market Watch" PDF (linked from `pse.com.ph/market-report/`; the link itself is discovered from that page's HTML, since the PDF filename encodes a week number that isn't otherwise predictable). The PDF's raw text stream interleaves the table with footer/disclaimer content rather than reading row-by-row, so `parseMarketWatchPdf.ts` does position-based extraction with `pdfjs-dist` (text items grouped into rows by y-coordinate, ordered into columns by x-coordinate) instead of a naive text regex.
  - **This lives in `etl/lib/`, deliberately *not* in `packages/sources/foreign-flow`.** `pdfjs-dist` has a real module-init side effect (a `DOMMatrix` polyfill check) that isn't tree-shakeable once re-exported through a package's barrel `index.ts` — putting it in the shared `foreign-flow` package broke static generation for *every* `apps/web` route (silent `DOMMatrix`/`Path2D` warnings during `next build`, confirmed by testing) even though `apps/web` never calls that code. Any future source needing a heavy/native-touching dependency (PDF parsing, image processing, etc.) belongs in `etl/lib/` or its own narrowly-scoped package — never in a `packages/sources/*` package whose barrel `apps/web` already imports for its types/mock. Pinned to `pdfjs-dist@4.0.379` specifically (not the latest) — newer majors require Node ≥22, while this repo's workflows and local tooling are on Node 20.
- **`packages/config`** (`@pseye/config`) — shared base `tsconfig.json` extended by every other package.
